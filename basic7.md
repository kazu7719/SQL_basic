## Sequel AceからSQL文を実行するメリット
・文末に;を付ける必要がない

# WHERE句
・・・取得するデータを条件で制限できる

WHERE <条件>

※条件には比較演算子を使用することができ、内容が文字列・日付は""をつける

# LIKE句
・・・WHERE句と合わせて文字列探索が可能

WHERE <列名> LIKE "検索文字"

※曖昧検索として%というワイルドカード使用可能

SELECT *
FROM users
WHERE prefecture
LIKE '%都'

# AND演算子
・・・複数の条件を指定して式を作り、その条件が全て正である時、式が正になる

WHERE 《a》 AND 《b》

# OR演算子
・・・複数の条件を指定して式を作り、その条件どちらか一つ正となれば式が正に

WHERE <a> AND <b>

# NOT演算子
・・・その条件に当てはまらないものを取得

WHERE NOT <a>

# BETWEEN演算子
・・・1つのカラムに対して上限と下限を指定して式を作り、カラムの値がその範囲に含まれる時、その式は正になるという演算子

WHERE <カラム名>　BETWEEN <下限>　AND <上限>

# IN演算子
・・・1つのカラムの対しリストを指定して、カラムの値がそのリストに含まれる時正になる

WHERE <カラム名> IN (値1, 値2, ....)
