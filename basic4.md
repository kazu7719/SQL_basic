### データの検索方法

## WHERE句
・・・取得するデータを「条件」によって制限できる句
WHERE句では取得するレコードの条件を指定し、その条件が正のレコードを取得する

WHERE <条件>

WHERE句の条件には、以下のように"="や"<=" のような比較演算子を用いる

WHERE id = 1

WHERE last_name = "阿部"

WHERE id <= 5


## LIKE句
・・・WHERE句と併用して使うことで文字列検索を行うことができる句

WHERE 《列名》 LIKE '検索文字'

曖昧検索として%というワイルドカードも使用可能

SELECT *
FROM users
WHERE prefecture
LIKE '%都'


## AND演算子
・・・複数の条件を指定して式を作り、その条件が全て正である時、式が正になる

WHERE 《a》 AND 《b》

SELECT *
FROM users
WHERE age <= 22 AND prefecture = "神奈川県"


## OR演算子
・・・複数の条件を指定して式を作り、その条件のうちどれかひとつ以上が正となれば、式が正になる

WHERE 《a》 OR 《b》

SELECT *
FROM users
WHERE age <= 20 OR prefecture = "東京都"


## NOT演算子
・・・1つの条件を指定して式を作り、その条件に当てはまらないものを取得する

WHERE NOT 《a》


## BETWEEN演算子
・・・カラムに対し上限と下限を指定して式を作り、カラムの値がその範囲内である場合、その式は正になる

WHERE 《カラム名》 BETWEEN 《下限》 AND 《上限》

## IN演算子
・・・1カラムに対しリストを指定して、カラムの値がそのリストに含まれている時、その式は正になる
WHERE 《カラム名》 IN (値1, 値2, ……)


## JOIN句
・・・指定したテーブルのカラムに登録された値が一致するデータをもとに、各テーブルのレコードを結合できる。
JOIN句はFROM句のあとに記述し、結合の対象となるテーブルを指定する。そして、結合する元となるカラムを、ONの後に=を用いて指定する

FROM 《テーブル名1》
JOIN 《テーブル名2》 ON 《テーブル名1.カラム名1》 = 《テーブル名2.カラム名2》

## CONCAT関数
・・・複数の文字列を連結できる関数

CONCAT(文字列1, 文字列2, ……)

## GROUP BY句
・・・ある特定のデータをグループ化するために用いる句
グループ化したデータは他の関数などと組み合わせて、グループごとの平均値や総量などを集計可能

GROUP BY カラム名

## COUNT関数
・・・データの総量をカウントするための関数。カラムを指定することで、カラムのデータの行数を取得可能

SELECT COUNT(カラム名)
